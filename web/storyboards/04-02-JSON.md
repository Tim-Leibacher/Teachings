## Szene 2: JSON-Grundstruktur – Objekte (1:20 Min.)

### Sprechertext
"JSON besteht hauptsächlich aus zwei Strukturen: Objekte und Arrays. Beginnen wir mit Objekten. Ein JSON-Objekt sieht fast wie ein JavaScript-Objekt aus – mit geschwungenen Klammern und Schlüssel-Wert-Paaren. Der Unterschied: In JSON müssen Schlüssel immer in doppelten Anführungszeichen stehen. Schauen wir uns ein Beispiel an: Hier ist ein Projekt-Objekt mit verschiedenen Eigenschaften. Beachte die doppelten Anführungszeichen bei den Schlüsseln und die Kommas zwischen den Paaren. Wichtig: Nach dem letzten Paar kommt kein Komma."

### Bildschirmdarstellung
VS Code mit `projekt.json`:

```json
{
  "id": 1,
  "titel": "Portfolio-Website",
  "beschreibung": "Meine persönliche Portfolio-Seite",
  "technologien": ["HTML", "CSS", "JavaScript"],
  "status": "in Arbeit",
  "fortschritt": 75,
  "istSichtbar": true,
  "startDatum": "2025-01-15"
}
```

Visualisierung daneben:
```
Objekt = { }
  ↓
Schlüssel-Wert-Paare
  ↓
"schlüssel": wert
```

### Hinweise
- Syntax-Highlighting aktivieren (JSON-Modus in VS Code)
- Markiere nacheinander:
  1. Geschwungene Klammern (orange)
  2. Schlüssel in Anführungszeichen (blau)
  3. Doppelpunkt als Trenner (rot)
  4. Werte unterschiedlicher Typen (grün)
- Zeige häufigen Fehler: `{titel: "Test"}` (ohne Anführungszeichen) → ✗

## Szene 3: JSON-Datentypen (1:10 Min.)

### Sprechertext
"JSON unterstützt sechs Datentypen. Erstens: Strings – Text in doppelten Anführungszeichen. Zweitens: Numbers – Zahlen ohne Anführungszeichen, mit oder ohne Dezimalstellen. Drittens: Booleans – true oder false, kleingeschrieben. Viertens: Arrays – Listen in eckigen Klammern. Fünftens: Objekte – verschachtelte Strukturen. Und sechstens: null – für fehlende Werte. Wichtig: undefined, Funktionen und Kommentare sind in JSON nicht erlaubt."

### Bildschirmdarstellung
Split-Screen mit Beispielen:

```json
{
  "string": "Das ist Text",
  "zahl": 42,
  "dezimalzahl": 3.14,
  "boolean": true,
  "array": [1, 2, 3],
  "objekt": { "nested": "value" },
  "leerWert": null
}
```

Daneben Tabelle:
```
Datentyp    | Beispiel           | Gültig
------------|-------------------|--------
String      | "Text"            | ✓
Number      | 42, 3.14          | ✓
Boolean     | true, false       | ✓
Array       | [1, 2, 3]         | ✓
Object      | {"key": "val"}    | ✓
null        | null              | ✓
undefined   | undefined         | ✗
Function    | function() {}     | ✗
```

### Hinweise
- Farbcodierung: Strings (grün), Numbers (blau), Booleans (orange)
- Zeige Gegenüberstellung: JavaScript vs. JSON
- Betone: JSON ist strikter als JavaScript

## Szene 6: JSON in JavaScript – parse() und stringify() (1:30 Min.)

### Sprechertext
"In JavaScript arbeitest du mit zwei wichtigen Funktionen: JSON punkt parse wandelt einen JSON-String in ein JavaScript-Objekt um – damit kannst du auf die Daten zugreifen und sie verwenden. JSON punkt stringify macht das Gegenteil: Es wandelt ein JavaScript-Objekt in einen JSON-String um – das brauchst du zum Speichern oder Versenden. Schauen wir uns beide Richtungen an."

### Bildschirmdarstellung
Browser DevTools Konsole:

```javascript
// JSON-String (von API oder Datei)
let jsonString = '{"name": "Sarah", "age": 18}';
console.log(typeof jsonString);  // "string"

// JSON parsen → JavaScript-Objekt
let person = JSON.parse(jsonString);
console.log(typeof person);      // "object"
console.log(person.name);        // "Sarah"

// JavaScript-Objekt → JSON-String
let projekt = {
  titel: "Portfolio",
  status: "in Arbeit"
};

let jsonOutput = JSON.stringify(projekt);
console.log(jsonOutput);
// '{"titel":"Portfolio","status":"in Arbeit"}'

// Mit schöner Formatierung (Einrückung)
let jsonFormatiert = JSON.stringify(projekt, null, 2);
console.log(jsonFormatiert);
/*
{
  "titel": "Portfolio",
  "status": "in Arbeit"
}
*/
```

Visualisierung:
```
JSON-String <--[parse()]--> JavaScript-Objekt
            <-[stringify()]-
```

### Hinweise
- Zeige Live-Ausgabe in der Konsole
- Betone typeof-Unterschied (string vs. object)
- Zeige stringify mit null, 2 für Formatierung
- Erkläre: Parse = Lesen, Stringify = Schreiben

## Szene 7: Häufige Fehler in JSON (1:20 Min.)

### Sprechertext
"JSON ist strikt – kleine Fehler machen die ganze Datei ungültig. Die häufigsten Fehler: Einfache statt doppelte Anführungszeichen, fehlendes Anführungszeichen beim Schlüssel, Komma nach dem letzten Element, oder trailing Comma genannt. Diese Fehler führen zu Parse-Errors. Zum Glück gibt es JSON-Validators, die dir helfen, Fehler zu finden. In VS Code siehst du Fehler direkt mit Rot-Wellenlinie."

### Bildschirmdarstellung
Split-Screen: Falsch vs. Richtig

**Falsch:**
```json
{
  'name': 'Sarah',        // ✗ Einfache Anführungszeichen
  alter: 18,              // ✗ Schlüssel ohne Anführungszeichen
  skills: ["HTML", "CSS",], // ✗ Komma nach letztem Element
  beruf: undefined        // ✗ undefined nicht erlaubt
}
```

**Richtig:**
```json
{
  "name": "Sarah",
  "alter": 18,
  "skills": ["HTML", "CSS"],
  "beruf": null
}
```

Zeige Browser-Konsole mit Parse-Error:
```javascript
JSON.parse('{name: "Test"}');
// Uncaught SyntaxError: Unexpected token n in JSON at position 1
```

Zeige JSON-Validator (jsonlint.com) mit Fehlermarkierung

### Hinweise
- Markiere Fehler rot
- Zeige Korrektur grün
- Verwende echten Parse-Error aus Konsole
- Tipp einblenden: "Nutze JSON-Validator beim Testen"

## Szene 8: Praktisches Beispiel – Portfolio-Daten laden (1:30 Min.)

### Sprechertext
"Schauen wir uns ein praxisnahes Beispiel an. Du hast eine JSON-Datei mit deinen Portfolio-Projekten und möchtest diese auf deiner Website anzeigen. Zuerst lädst du die JSON-Datei mit fetch, dann parsest du die Daten automatisch mit der json-Methode, und schliesslich durchläufst du die Projekte und generierst HTML. So funktionieren moderne Web-Apps: Daten kommen im JSON-Format, JavaScript verarbeitet sie und baut daraus die Benutzeroberfläche."

### Bildschirmdarstellung
VS Code mit drei Dateien:

**projekte.json:**
```json
{
  "projekte": [
    {
      "id": 1,
      "titel": "Portfolio-Website",
      "beschreibung": "Meine persönliche Seite",
      "technologien": ["HTML", "CSS", "JavaScript"]
    },
    {
      "id": 2,
      "titel": "To-Do App",
      "beschreibung": "Aufgabenverwaltung",
      "technologien": ["JavaScript"]
    }
  ]
}
```

**script.js:**
```javascript
// JSON-Datei laden
fetch('projekte.json')
  .then(response => response.json())  // Automatisches Parsen
  .then(data => {
    console.log('Geladene Daten:', data);
    
    // Durch Projekte iterieren
    data.projekte.forEach(projekt => {
      console.log(`Projekt: ${projekt.titel}`);
      console.log(`Tech: ${projekt.technologien.join(', ')}`);
    });
  });
```

**Browser-Ausgabe (Konsole):**
```
Geladene Daten: {projekte: Array(2)}
Projekt: Portfolio-Website
Tech: HTML, CSS, JavaScript
Projekt: To-Do App
Tech: JavaScript
```

### Hinweise
- Zeige Dateistruktur im Explorer (projekte.json neben index.html)
- Live-Demo: Code ausführen und Konsole zeigen
- Betone: response.json() macht automatisch JSON.parse()
- Hinweis: Später mehr zu fetch und Promises

## Szene 9: JSON vs. JavaScript-Objekt (0:50 Min.)

### Sprechertext
"Zum Abschluss nochmal der wichtige Unterschied: JSON ist ein Textformat – ein String. JavaScript-Objekte sind Datenstrukturen im Speicher. JSON kann gespeichert, übertragen und von verschiedenen Programmiersprachen gelesen werden. JavaScript-Objekte existieren nur während dein Code läuft. Deshalb konvertierst du: Von JSON-String zu Objekt zum Arbeiten, von Objekt zu JSON-String zum Speichern oder Senden."

### Bildschirmdarstellung
Vergleichstabelle:

```
Eigenschaft        | JavaScript-Objekt    | JSON
-------------------|---------------------|------------------
Format             | Datenstruktur       | Text (String)
Schlüssel          | Mit/ohne ""         | Immer mit ""
Funktionen         | Erlaubt             | Verboten
undefined          | Erlaubt             | Verboten
Kommentare         | Erlaubt             | Verboten
Verwendung         | Im Code arbeiten    | Speichern/Senden
Konvertierung      | JSON.stringify()    | JSON.parse()
```

Beispiel nebeneinander:
```javascript
// JavaScript-Objekt
let obj = {
  name: "Sarah",        // Ohne "" erlaubt
  greet: function() {   // Funktion erlaubt
    console.log("Hi");
  }
};

// JSON (String)
let json = '{"name": "Sarah"}';  // Nur Daten, keine Funktionen
```

### Hinweise
- Tabelle mit BAND-Farben gestalten (Blau 700 für Header)
- Zeige Unterschiede fett markiert
- Betone: JSON = Datenaustauschformat, nicht Programmiersprache
