# Storyboard HTML 3 – Formulare

| Punkt                                      | Sprechertext                                                                                                                                      | Bildschirmdarstellung / Beispiel                                                           | Dauer    |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ | -------- |
| **1. Einstieg: Warum Formulare?**          | „Formulare sind das Tor zur Interaktion im Web. Jede Anmeldung, jede Suche, jede Bestellung – all das funktioniert mit HTML-Formularen."         | Zeige bekannte Formulare: Google-Suche, Login-Seite, Kontaktformular.                     | 0:30 min |
| **2. Das form-Element**                    | „Jedes Formular beginnt mit dem `<form>`-Tag. Es umschliesst alle Eingabefelder und bestimmt mit `action` und `method`, wohin die Daten gehen."  | Zeige ein einfaches `<form>`-Element im Code-Editor.                                       | 0:40 min |
| **3. Textfelder mit input**                | „Mit `<input type="text">` erstellst du ein Textfeld. Der `name`-Wert identifiziert das Feld, und `placeholder` zeigt einen Hinweis."            | Zeige Textfeld im Code und im Browser mit Platzhaltertext.                                 | 0:40 min |
| **4. Labels für Barrierefreiheit**         | „Jedes Eingabefeld braucht ein `<label>`. Das `for`-Attribut verbindet Label und Feld – wichtig für Screenreader und Nutzerfreundlichkeit."      | Zeige Label-Input-Kombination, klicke auf Label, Input wird fokussiert.                   | 0:35 min |
| **5. Verschiedene Input-Typen**            | „Es gibt viele Input-Typen: `email` zeigt die passende Tastatur auf Smartphones, `password` verbirgt die Eingabe, `number` erlaubt nur Zahlen."  | Zeige verschiedene Input-Typen im Browser und auf einem Smartphone-Simulator.             | 0:40 min |
| **6. Checkboxen und Radio-Buttons**        | „Mit `type="checkbox"` können mehrere Optionen gewählt werden. `type="radio"` erlaubt nur eine Auswahl pro Gruppe mit gleichem `name`."          | Zeige Checkboxen für Hobbys und Radio-Buttons für Geschlecht oder Zahlungsart.            | 0:40 min |
| **7. Dropdown-Menüs mit select**           | „Mit `<select>` und `<option>` erstellst du Auswahlmenüs. Das spart Platz und macht Formulare übersichtlicher."                                  | Zeige ein Dropdown-Menü für Länder oder Kategorien.                                        | 0:30 min |
| **8. Textbereiche mit textarea**           | „Für längere Texte gibt es `<textarea>`. Hier können Nutzer Nachrichten oder Kommentare eingeben."                                               | Zeige ein Kontaktformular mit Nachrichtenfeld.                                             | 0:25 min |
| **9. Buttons und Absenden**                | „Mit `<button type="submit">` sendest du das Formular ab. `type="reset"` setzt alle Felder zurück."                                              | Zeige beide Button-Typen und deren Funktion im Browser.                                    | 0:30 min |
| **10. Pflichtfelder mit required**         | „Das `required`-Attribut macht ein Feld zum Pflichtfeld. Der Browser prüft automatisch, ob es ausgefüllt ist, bevor das Formular abgesendet wird." | Zeige Validierung beim Absenden eines unvollständigen Formulars.                          | 0:30 min |
| **11. Zusammenfassung & Best Practices**   | „Ein gutes Formular ist übersichtlich, hat klare Labels und fragt nur nötige Informationen ab. Nutze die richtigen Input-Typen für bessere UX."  | Zeige ein vollständiges, sauber strukturiertes Formular mit Einrückung.                   | 0:30 min |
